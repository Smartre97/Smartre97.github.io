# 创建版本控制

打开你准备进行版本控制 / 多人协同的虚幻 5 工程项目
（如果担心不正常的操作造成文件丢失，请先提前备份项目或开一个新项目用于实验）。

点击右下角 “版本控制”-“连接到版本控制”
提供方选择 Git（beta 版）
Git 路径选择为 Git.exe 的路径（一般在你下载的 Git 路径目录下的 bin 文件夹中）
勾选 “添加一个.Gitattributes 文件启用 Git LFS”
点击 “用 Git 初始化项目”

# 提交文件

可点击右下角 “版本控制”，选择 “提交内容”，此操作会将所有文件提交到本地仓库。
之后的每次修改可选择 “签出已修改文件”

# Github 配置

# 创建一个仓库

你可以在左侧挡泥板的 “Top Repositories” 右边点击 New 以创建你的第一个空白远程仓库
此时可自由编辑名称、描述、是否对外开放，“Add .gitignore” 可选择为 “UnrealEngine”
（私人项目一般不建议对外开放，关于协同可以稍后为队友开放权限）

# 创建 ssh 连接

由于国内 https 连接时长会无法正常范围，导致项目无法正确提交到云端或下载
此时则需要使用 ssh 进行连接

# 进入 ssh 文件夹

打开 Git bash

此时会生成一大串内容，那就是你的 ssh key，将其全部复制

# 在 Github 中进行配置

进入 Github - 设置 - SSH and GPG keys
选择 New key
title 名字随便取，将刚刚复制的 ssh key 粘贴到下方空白处
注：不要随意泄露你的 ssh key

# 验证是否配置成功

# 汉化

# 创建分支

多人协同时的工作不建议在主分支上进行，可以在 Current branch 中选择创建一个新的分支
工作确定完成时，可在此选择下方选择合并一个分支到 main

# 同步与推送

可以在上方导航栏中选择 Repository，选择推送当前分支到 github 或同步 github 的项目到自己的分支
也可以直接点击 Publish branch 上传当前分支到 github
